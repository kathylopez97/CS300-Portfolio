# CS300-Portfolio
CS 300 Data Structures and Algorithms Portfolio including Project One analysis and Project Two C++ implementation.

## CS 300 Data Structures and Algorithms Portfolio

## Project Overview

This portfolio showcases the projects I completed in CS 300: Data Structures and Algorithms. Throughout this course, I analyzed multiple data structures and implemented an advising assistance program for ABC University. The application reads course information from a file, validates prerequisite relationships, stores the data in a binary search tree, prints courses in alphanumeric order, and allows users to search for individual courses. These projects strengthened my understanding of algorithmic efficiency, runtime analysis, software design, and the impact of data structure selection on application performance.

## Project One

Project One focused on evaluating vectors, hash tables, and binary search trees to determine the most effective data structure for the advising assistance program. I analyzed the runtime complexity, memory requirements, advantages, and limitations of each option before selecting the binary search tree as the recommended solution. While hash tables provide fast average search performance, they do not maintain sorted data. A binary search tree provides efficient searching while naturally preserving alphanumeric order through in-order traversal, making it the best overall solution for the project requirements.

## Project Two

Project Two transformed the design from Project One into a fully functional C++ application. The program reads and validates course data from a file, verifies prerequisite relationships, stores course information in a binary search tree, displays all courses in alphanumeric order, and allows users to retrieve detailed information for individual courses. The implementation emphasizes modular programming, efficient searching, input validation, and clean code organization to produce a reliable and maintainable application.

## Reflection

## What was the problem you were solving in the projects for this course?

The objective was to design and implement an advising-assistance program that enables academic advisors to efficiently organize, search for, and display course information. The solution needed to support fast lookups while maintaining courses in alphabetical order and accurately displaying prerequisite relationships as the course catalog grows.

 ## How did you approach the problem? Consider why data structures are important to understand.

I approached the project by first comparing multiple data structures before writing any code. Evaluating vectors, hash tables, and binary search trees helped me understand how data structure selection directly affects runtime performance, memory usage, scalability, and maintainability. Based on this analysis, I selected the binary search tree because it provides efficient searching while naturally maintaining sorted course data. This experience reinforced that choosing the appropriate data structure is one of the most important decisions in software design.

## How did you overcome any roadblocks you encountered while going through the activities or project?

The most significant challenges involved implementing binary search tree operations, validating prerequisite relationships, handling file input, and resolving compiler errors. I overcame these challenges by breaking the program into smaller components, testing each function independently, debugging systematically, and refining the implementation through repeated testing. This incremental approach improved both the reliability and accuracy of the final application.

## How has your work on this project expanded your approach to designing software and developing programs?

This project strengthened my understanding of the software design process by demonstrating the value of planning before implementation. Designing algorithms, evaluating data structures, and analyzing runtime complexity before writing code resulted in a cleaner architecture and a more efficient solution. I now place greater emphasis on thoughtful design decisions that improve both performance and long-term maintainability.

## How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This project improved my ability to write organized, maintainable, and adaptable software by encouraging modular program design, descriptive naming conventions, meaningful documentation, and consistent formatting. Separating the application into reusable functions simplified testing and debugging while making the code easier to understand and extend. These practices will continue to guide the way I develop software in future academic and professional projects.
